Findings

- P1: Portfolio context can leak project data outside project membership. src/lib/ai-assistant.ts loads every non-archived project in the active organization, while the tools and project conversations correctly check members: { some: { userId } } elsewhere. In a portfolio conversation, a user can get names, IDs, health, PPC, roadblocks, and dates for projects they may not belong to before any tool permission boundary applies. Fix by passing userId into buildAssistantContext and filtering projects by membership, matching src/lib/assistant-tools.ts.

- P2: Deterministic tool failures persist as blank assistant turns. In the direct deterministic path, the catch block writes only tool-output-error and finish, then onEnd persists messageText(responseMessage).trim(). That means permission errors, no-op proposals, or parser failures can become empty assistant messages in history. Add a text part with the error message, or skip persistence for failed deterministic tool runs.

- P2: Context and tool policy fight each other. Each model turn builds full portfolio context plus detailed focus-project context, then instructs the model to call a read-only tool for every current-data question anyway. This is costly, increases stale-context conflict risk, and leaves only six steps for forced tools plus follow-up calls. Prefer a smaller scoped briefing plus tool-first retrieval.

What's Strong

The write path is sensibly confirmation-gated. Proposals snapshot state, expire after 30 minutes, and schedule confirmations use serializable transactions and re-check affected tasks before mutation. That lines up well with current agent guidance: simple loops, clear tool contracts, environment feedback, and human checkpoints.

Principles Check

Current agent guidance from OpenAI and Anthropic emphasizes tool ergonomics, scoped context, permission boundaries, and multi-turn evals. Anthropic's agent eval guidance specifically calls out tool calls, state mutation, transcripts, and final environment state as evaluation targets; their tool guidance recommends measuring tool errors, redundant calls, and invalid parameters. Your runtime has good deterministic safeguards, but needs an eval suite for full agent trajectories: permission-scoped portfolio Q&A, document search grounding, forced-tool clarifications, failed proposal tools, and "confirm/yes" recovery.
